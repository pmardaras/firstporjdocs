# Requisitos del Proyecto: Sistema de Reservas

## 1. Objetivo
El sistema permitirá gestionar reservas de servicios de manera eficiente, tanto para clientes como para el personal.

## 2. Requisitos Funcionales
- **RF001:** El usuario podrá crear una nueva reserva.
- **RF002:** El usuario podrá modificar una reserva existente.
- **RF003:** El usuario podrá cancelar una reserva.
- **RF004:** El sistema enviará notificaciones por email al usuario al confirmar o cancelar una reserva.

## 3. Requisitos No Funcionales
- **RNF001:** El sistema debe responder en menos de 2 segundos por acción.
- **RNF002:** La interfaz debe ser accesible desde móvil y desktop.
- **RNF003:** Todos los datos deben almacenarse de forma segura, cumpliendo con GDPR.

## 4. Prioridad
- Alta: RF001, RF004  
- Media: RF002, RF003  
- Baja: RNF002, RNF003
